# Experiments

This folder holds one subfolder per open-source project studied. Each project runs the same four pipeline configurations (C1–C4) on its own GitHub fork, and the measurement outputs are stored here.

---

## Projects

| # | Folder | Project | Language | GitHub Fork | Status |
|---|---|---|---|---|---|
| 01 | `project-01-httpie-cli/` | HTTPie CLI | Python | [Umer-2612/httpie-cli-carbon-study](https://github.com/Umer-2612/httpie-cli-carbon-study) | Pilot runs done, 30-run protocol in progress |

---

## The Four Configurations

These are defined once and applied to every project. The YAML implementation differs by language but the strategy is the same.

| Config | Branch name | What changes | Strategy |
|---|---|---|---|
| **C1** | `experiment/c1-baseline` | Nothing, Eco-CI added for measurement only | Baseline |
| **C2** | `experiment/c2-pip-cache` | Dependency caching enabled (`cache: pip` / `npm` / `maven`) | Caching only |
| **C3** | `experiment/c3-consolidation` | All separate workflow files merged into one | Consolidation only |
| **C4** | `experiment/c4-combined` | C3 + caching + path-based trigger filters | All three combined |

---

## Adding a New Project

1. Fork the target repo to `github.com/Umer-2612/[project-name]-carbon-study`
2. Create the four experiment branches with Eco-CI instrumented in the workflows
3. Create `experiments/project-NN-[name]/` here with the same folder structure as `project-01-httpie-cli/`
4. After running 30 × 4 = 120 triggered runs, download the Eco-CI artifacts using `scripts/collect_results.py`
5. Store results in `results/raw_data.csv` and run `analysis/energy_analysis.ipynb`

## Folder Structure per Project

```
project-NN-name/
├── .github/workflows/        # instrumented YAML files (reference copies)
├── analysis/
│   └── energy_analysis.ipynb # Wilcoxon, Cliff's delta, SCI calculations
├── results/
│   ├── raw_data.csv          # Eco-CI measurements downloaded from GitHub
│   └── figures/              # charts generated by notebook
├── scripts/
│   └── collect_results.py    # downloads and parses GitHub Actions artifacts
├── README.md                 # project summary, pilot results, how to run
└── RESEARCH_LOG.md           # audit notes, bugs found and fixed, decisions
```
